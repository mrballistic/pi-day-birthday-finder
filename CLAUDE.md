## Project Overview

Pi Day Birthday Finder — a single-page app that finds the user's birthday in the first 5M digits of π, with a dramatic scanning animation and celebration reveal.

## Commands

```bash
# Development
npm run dev

# Production build + start
npm run build && npm start

# Lint
npm run lint

# Regenerate pi digits (if public/pi-digits.json is missing)
node scripts/generate-pi.js
```

## Stack

- **Next.js 15** (App Router, TypeScript, `src/` directory)
- **MUI v6** (`@mui/material`, `@mui/x-date-pickers`) with Emotion
- **Framer Motion** for transitions and `AnimatePresence` phase switching
- **canvas-confetti** for celebration effects (dynamically imported)
- **Google Fonts:** Inter (body, `--font-inter`) + JetBrains Mono (digits, `--font-mono`)
- **date-fns** for MUI DatePicker adapter

## Architecture

### Three-Phase State Machine (`src/app/page.tsx`)

The app is a client-side state machine cycling through three phases:

1. **Phase 1 — Input** (`BirthdayInput`): Date picker + "Search π" CTA
2. **Phase 2 — Scan** (`ScanAnimation` + `DigitStream` + `PiTriviaTicker`): 6-second animated digit scan with speed ramping (slow → fast → dramatic slowdown near match). Animation is purely cosmetic — the actual `indexOf` search is instant.
3. **Phase 3 — Result** (`ResultCard` + `ConfettiExplosion`): Celebration with confetti, animated position counter, digit context strip, share button

Phases transition via `AnimatePresence mode="wait"`. The scan-to-result transition is triggered by `useScanAnimation` completing.

### Search Strategy (`src/lib/searchStrategies.ts`)

Searches pi digits in priority order, longest match first: MMDDYYYY → DDMMYYYY → MMDDYY → DDMMYY → MMDD → MD. Returns the first match found. A 2-digit match is guaranteed within 5M digits.

### Scan Animation (`src/hooks/useScanAnimation.ts`)

Uses `requestAnimationFrame` with eased progress: slow start (0-15%), fast middle (15-70%), dramatic slowdown (70-100%). Duration is always ~6 seconds regardless of actual match position. Respects `prefers-reduced-motion` by skipping directly to result.

### Pi Data

- `public/pi-digits.json` — 5M digits after the decimal as `{"digits":"14159..."}` (~4.8MB)
- Loaded async via `fetch()` on mount, cached in a ref (`src/lib/piLoader.ts`)
- Generated by `scripts/generate-pi.js`

### Theme

- Dark cosmic theme: `#0a0a1a` → `#1a1a2e` → `#16213e` gradient
- Neon accent colors defined as CSS custom properties in `globals.css`
- MUI dark theme configured in `src/theme/theme.ts`, provided via `src/app/ThemeRegistry.tsx` (client component wrapper)

## Key Design Decisions

- `canvas-confetti` is dynamically imported to avoid bloating the initial bundle
- `FloatingDigits` uses raw canvas for ambient particle background (not React rendering)
- The trivia ticker shuffles facts with Fisher-Yates on each scan and cross-fades every 3 seconds
- All animations use GPU-accelerated properties (`transform`, `opacity`)
- `aria-live` regions announce scan state for screen readers
